Building an MCP server to search our ITGlue documentation

6 June 2026 · MCP · Python · SQLite
MCPPythonSQLiteITGlueCopilot Studio

Recently I've been working through Anthropic's Academy courses on the Model Context Protocol (MCP), and realised I already had a use case in front of me to apply what I'd learnt.

The MSP I work at has a lot of knowledge stored in our documentation platform (ITGlue), but as a new starter it could be hard to find what I was looking for.

Syncing ITGlue into something searchable

I started by writing a Python sync layer that pulls from ITGlue's REST API and stores the data in a local SQLite database with full-text search: both the structured records (network, licensing, etc.) and the hand-written documents/procedures, plus the list of client organisations. Crucially, I scoped the sync so it never pulls the passwords stored in ITGlue. Learning ITGlue's data model properly was important here to avoid 404 errors (it took me a while to realise the /documents endpoint didn't exist)

Learning ITGlue's data model properly was important here to avoid a stream of 404 errors — it took me a while to realise the /documents endpoint didn't exist, and that documents live under a different part of the API than I'd assumed.

Exposing it through MCP

On top of that database I built the MCP server, exposing a small set of read-only tools: search the documentation, fetch a full document, and look up an organisation.

Getting Microsoft Copilot Studio to actually talk to my server helped solidify what I'd learnt in Anthropic's courses. Stateless HTTP was the easiest way to set up communication between the MCP server and MCP client in my case.

Worth it

Once it was all connected, it was quite rewarding to drop it straight into my workflow and start finding information far more quickly. It's a small tool, but it turned an abstract course on a new protocol into something I now use most days.
